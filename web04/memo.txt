웹 서버가 실행시키는 프로그램 : 웹 애플리케이션

웹 서버와 웹 애플리케이션 사이에 데이터를 주고받기 위한 규칙 : CGI(Common Gateway Interface)

자바로 만든 웹 애플리케이션 : 서블릿 : Server + Applet의 합성어
 : 클라이언트에게 서비스를 제공하는 작은 단위의 서버 프로그램

서블릿 컨테이너 : 서블릿의 생성에서 실행, 소멸까지 서블릿의 생명주기를 관리하는 프로그램

서블릿을 만들 때에는 반드시 Servlet 인터페이스를 구현해야만 함

서블릿을 만들 때 Servlet 인터페이스를 직접 구현하는 것보다 GenericServlet 클래스를 상속받으면 좀 더 쉽고 편리하게 개발할 수 있음
Why ? service()를 제외한 나머지 메서드를 모두 구현하여 service() 구현에만 집중할 수 있음

--------

HttpServlet 클래스를 이용하여 서블릿 만들기, 회원 목록 조회 구현

CRUD : Create, Retrieve, Update, Delete

자바에서는 데이터베이스와의 통신을 위해 JDBC 기술을 제공함

작업 결과를 출력한 후 다른 페이지로 이동할 때는 리프래시 사용, 
작업 결과를 출력하지 않고 다른 페이지로 이동할 때는 리다이렉트로 처리

GET : 데이터 조회나 삭제 요청처럼 간단한 데이터를 보내는 경우
POST : 데이터 등록이나 변경, 로그인과 같은 대량의 데이터를 보내거나 브라우저의 주소창에 노출되지 말아야 할 데이터를 보내는 경우

HttpServlet을 상속받아서 서블릿을 만들면 GET과 POST요청을 쉽게 구분하여 처리할 수 있음.
service()를 오버라이딩 하는 대신에 doGet()과 doPost()를 재정의

한글 데이터가 깨짐 : 서블릿에서 데이터를 꺼낼 때 클라이언트가 보낸 데이터를 유니코드로 변환하기 때문
getParameter()를 호출하여 데이터를 꺼내기 전에 setCharacterEncoding()을 호출하여 클라이언트가 보낸 데이터가 UTF-8임을 먼저 지정

특정 서블릿에서 사용할 정보라면 서블릿 초기화 매개변수를 사용, 여러 서블릿에서 공통으로 사용할 정보라면 컨텍스트 초기화 매개변수를 사용

서블릿을 실행하기 전이나 후에 특별한 작업을 수행해야 한다면 필터를 사용
